[技巧] 多階段建構

1. 本機上先試試

  • 這邊來舉個 C 語言的例子。
  • 假設我現在已經寫好一個 hello.c 的程序，內容如下：
    #include <stdio.h>

void main(int argc, char *argv[])
{
    printf("hello %s\n", argv[argc - 1]);
}
  • 那我在本機，可以用 gcc ，把這個 hello.c ，編譯成一個執行檔，叫 hello ，作法如下：
    gcc --static -o hello hello.c
  • 做完後，現在這個專案資料夾，就有兩個檔案了：
    ls
# hello hello.c
  • 那我執行這個 hello檔，並在後面放 argument，就可以得到我要的效果：
    ./hello docker
# hello docker
./hello world
# hello world
./hello friends
# hello friends

Docker化

  • 那假如，我今天想把這個應用，用 docker 來做
  • 我的 Dockerfile 就會長這樣：
    FROM gcc:9.4
COPY hello.c /src/hello.c
WORKDIR /src
RUN gcc --static -o hello hello.c
ENTRYPOINT [ "/src/hello" ]
CMD []

  • 可以發現，這個 image，竟然高達 1.14 GB。原因是 FROM gcc 時，就已經下載 1.14GB 下來了
  • 但實際上，我們編譯完後，我根本不需要 gcc 了。我有編譯好的 hello 後，我只要一個輕量的 linux，就可以執行這個程序了。
  • 那 Dockerfile 現在可以改寫成這樣：
# 階段一： 負責編譯用  
FROM gcc:9.4 AS builder
COPY hello.c /src/hello.c
WORKDIR /src
RUN gcc --static -o hello hello.c

# 階段二： 負責執行
FROM alpine:3.13.5
COPY --from=builder /src/hello /src/hello
ENTRYPOINT [ "/src/hello" ]
CMD []
  • 重點在
    • 階段一的第一行，我幫他取了個外號叫 builder ，所以第一塊做完後，可以把他想像成有一台 linux 電腦，把結果編譯出來了，結果放在 /src 下面，裡面有 hello 和 hello.c
    • 階段二我就只下載輕量化的 linux (alpine)，然後，從剛剛 builder 這台 linux 電腦中，copy他的 /src/hello ，到 alpine 這台 linux 的 src/hello 。
  • 那這樣最後的 image，就只會有階段二的 alpine 的結果。
