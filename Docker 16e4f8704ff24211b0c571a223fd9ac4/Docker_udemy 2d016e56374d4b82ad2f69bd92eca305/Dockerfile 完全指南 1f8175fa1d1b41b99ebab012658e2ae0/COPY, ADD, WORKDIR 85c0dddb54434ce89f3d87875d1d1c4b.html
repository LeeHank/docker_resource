COPY, ADD, WORKDIR

1. COPY and ADD

  • COPY 和 ADD 都是要把我本地的資料夾or檔案，複製到 image 裡面
  • 例如這樣：
    FROM python:3.9.5-alpine3.13
COPY hello.py /app/hello.py
  • 那意思就是，在 build image 的過程中，我要把我本地端的 hello.py ，給複製到 image 裡的 alpine 作業系統(因為他的 base image 看起來 OS 是用 alpine) 裡面的 /app/hello.py 這個位子。
  • 但兩者的差別在於：如果你要複製的文件是 "壓縮檔"，那你用 ADD 的時候，他會自動幫你解壓縮。例如：
FROM python:3.9.5-alpine3.13
ADD hello.tar.gz /app/
  • 那在 image 裡的 alopine3 作業系統中，就會先把 hello.tar.gz 給解壓縮後，再放到 /app/ 這個位子。
  • 這邊提醒一下，如果你只給路徑的話，最後一定要加斜線，不要寫成 /app ，因為這樣就不是進到 app 裡面，而是你複製的結果是一個叫 app 的資料夾。
  • 那，何時用 COPY 何時用 ADD 呢？ follow 這個原則： 需要解壓縮才用 ADD ，不然都用 COPY
  • 另外也提醒，不管你是複製資料夾或檔案到 image 裡面，他的權限都會繼承。例如，你剛剛 copy 的 hello.py ，如果原本在本機 linux 的權限只有可讀，那他被放到 image 的 alpine 裡面時，他的權限也是只有可讀。

2. WORKDIR

  • 沒講新東西，可看 hahow 那時的筆記 [這裏] ，或看以下簡單整理即可：
FROM python:3.9.5-alpine3.13
WORKDIR app
COPY hello.py hello.py
  • 那因為設定了 working directory 在 alpine 的 app 這裏，所以下一行的 copy，就會把文件放到 app/hello.py 這裏
