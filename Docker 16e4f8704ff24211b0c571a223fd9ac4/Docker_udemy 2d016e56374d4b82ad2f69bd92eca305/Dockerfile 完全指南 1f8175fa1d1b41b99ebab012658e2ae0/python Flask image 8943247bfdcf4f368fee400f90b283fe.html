1. 先在本機上做一個跑得起來的 flask

  • 我在以下路徑，做了一個 flask 專案： /Volumes/GoogleDrive/我的雲端硬碟/Docker_Udemy/flask_try
  • 在這資料夾中，我新增一個 app.py ，內容如下：
    from flask import Flask

app = Flask(__name__)  # 用Flask這個function，建立一個 web application 的 object，叫他 app
# Flask() 是一個函數，裡面的 argument 放 __name__
# __name__ 是 python 內建的變數，他會儲存目前這個程式在哪個模組下執行
# 如果此程式被當主程式來執行，那 __name__的結果就是 "__main__" 這個字串

# 用 decorator 來裝飾底下的 helloworld 函數
# 如何裝飾？就是當user發出GET request，要求查看 "/" 下的檔案時，我會執行helloworld這個函數，並把結果回傳給user


@app.route('/')
def helloworld():
    return "hello world"


if __name__ == "__main__":  # 如同上述，這邊就是在檢查 __name__ 這個變數的結果是不是 "__main__"
    app.run(
        host="0.0.0.0",  # 允許訪問的主機IP，設為 0.0.0.0 表示所有主機的意思，這樣所有主機都可以訪問我的 app
        port=5000
    )
  • 重點在最後三行，為了讓這個服務可以被外部主機訪問，host把它設為 0.0.0.0
  • 寫好這個 app.py 後，在 terminal 執行以下指令，就把這支 app 給起起來了：
    cd /Volumes/GoogleDrive/我的雲端硬碟/Docker_Udemy/flask_try
python app.py

  • 就如截圖上寫的，只要在 browser 上寫 http://0.0.0.0:5000 ，就可以看到網頁。

2. 寫成 image

  • 接下來，我們就把這個 網頁app，當成我要部署的應用。然後我想包成一個 docker image 來發布
  • 首先，寫 dockerfile，我就在這個專案資料夾下，新增一個 Dockerfile，內容如下：
FROM python:3.9.5-slim
RUN pip install flask
WORKDIR /src
COPY app.py ./
CMD python app.py
  • 可以發現，指令都蠻簡單的：
    • 我先下載 python 的 image
    • 安裝 flask 這個套件
    • 把工作目錄設在 /src
    • 把本機的 app.py ，複製到 container 裡的工作目錄下 (就是 /src 下)
    • 執行 python app.py 來開啟服務
  • 寫好後，就來 build image，在 terminal 裡面下：
    docker image build -t cktottti/docker_flask_try .
  • 寫好後，開 container 來跑他：
    docker container run -p 5678:5000 cktottti/docker_flask_try
  • 這邊要注意的，是 port 的對應，我讓本機的 5678，對到 container 裡的 5000 (因為 app.py 裡面是設定 port 為 5000)
  • 那就成功開啟服務了，看截圖：
