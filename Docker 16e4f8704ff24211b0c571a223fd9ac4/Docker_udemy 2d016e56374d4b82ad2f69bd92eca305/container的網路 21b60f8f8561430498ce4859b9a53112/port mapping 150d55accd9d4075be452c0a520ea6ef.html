port mapping

1. 在開 container 時，做 port mapping

  • 我們先看下面這張圖
  • 這張圖分左右兩塊，分別是主機1(host1) 和 主機2(host2)
  • 我在主機1 (host1)上，有兩個 container (nginx container, busybox)，這兩個 container 都被定義在 docker0 這個 bridge 網路之下
  • 所以，這兩個 container 如果要 ping 到 host2，那是 ok的，作法就是先透過 docker0 傳到 host1，再透過 NAT，把 IP 變成 192.168.200.10 (公網)，然後透過 HTTP 去訪問 host2
  • 但如果，我想從 host2 ，訪問 host1 裡面的 nginx container 這個 container，那目前是做不到的。因為他只會知道公網 IP (host1 的 192.168.200.10)，不會知道私網 IP (172.17.0.2)
  • 所以，要解決這個困境，就是做 port mapping，我在公網的地方設一個 port (e.g. 8080)，然後私網的 IP 也設一個 port (e.g. 80)，然後在建立 nginx container 這個 container 時，特別寫下： -p 8080:80 ，就知道 port 的 mapping是供網的 8080 對到私網的 80，那當我從 host2 要去訪問 host1 時，我只要敲 8080 這個 port ，他就會幫我連到私網的 80 這個 port 了。

2. 在 Dockerfile中， EXPOSE 一個 port

  • 那有時候也會看到，在 Dockefile 中，去寫 EXPOSE 80
  • 那，直接講結論，不寫EXPOSE 80 這句，完全沒差。因為，這句話實質上沒有做任何功能，只是把這個資訊寫進 image 裡面而已 (註解功能)。真的在做 port mapping 時，我們還是要在開 container 的時候去做。
  • 但如果在 Dockerfile 裡面有寫 ，那就像是有個註解的功能，讓使用你 image 的人知道，你的 image 需要做 port mapping。
  • Q: 為啥用你 image 的人會知道？
  • Ans: 因為當你把 image 包好後，例如就叫他 my-flask 這個 image 好了，你可以用 docker image inspect my-flask ，就可以看到這個 image 裡面的 Config 底下，有 ExposedPorts ，裡面就寫著你剛剛寫的 80 。所以，當 user 要用你的 image時，他就可以去查，有哪些 port ，我需要在開 container 時，做 port mapping。
