  • 今天的目標是這樣：
  • 我們想要開兩個 container:
    • flask-demo: 就是個 flask 的 image
    • redis-server:
  • 然後，我要架一個 demo-network 這個 bridge 網路。

1. 環境準備

  • 我會在 AWS EC2 linux server 上做事
  • 我在這個路徑( /Volumes/GoogleDrive/我的雲端硬碟/Docker_Udemy/flask_redis )，開了資料夾，放這次的專案
  • 這個專案資料夾裡面，放了：
    • app.py

  • 編輯好後，我再用 Filezilla 連到 AWS EC2 linux server，把專案資料夾 copy 過去 ( home/ubuntu/flask_redis )

1.1 app.py

  • app.py 的內容如下：
from flask import Flask
from redis import Redis
import os
import socket

app = Flask(__name__)
redis = Redis(host=os.environ.get('REDIS_HOST', '127.0.0.1'), port=6379)


@app.route('/')
def hello():
    redis.incr('hits')
    return f"Hello Container World! I have been seen {redis.get('hits').decode('utf-8')} times and my hostname is {socket.gethostname()}.\n"

if __name__ == "__main__":  # 如同上述，這邊就是在檢查 __name__ 這個變數的結果是不是 "__main__"
    app.run(
        host="0.0.0.0",  # 允許訪問的主機IP，設為 0.0.0.0 表示所有主機的意思，這樣所有主機都可以訪問我的 app
        port=5000
    )
  • 這邊注意一下， 中間的 redis = Redis() 這邊，他的 host 的 ip，是用 os.environ.get() ，去抓環境變數，哪個環境變數？ REDIS_HOST 這個環境變數。所以等等會需要把 REDIS_HOST 給定義好，內容就是一段 IP。那回頭看最上面的圖，這邊要寫的 ip，就是 redis-server 這個 container 被分配到的 ip。
  • 補充一下，因為要取得這個 container 的 ip，我們之前都是用互動模式進到這個 container 中，去用 ifconfig 來看。但這樣實在太累了。這邊教一招，如果 container 是在自己定義的 bridge network 之下的話，直接寫 container name，就等於寫 ip。
  • 例如，我在 flask-demo 這個 container 的互動模式中，我在 linux bash 裡面打 ping redis-server 可以直接 ping 到東西。
  • 所以，我的想法就是，我等等寫 Dockerfile 時，會做一個環境變數叫 REDIS_HOST ，然後在把 flask-demo 這個 container 開起來的時候，我要把 redis-server 這個已經開好的 container，當作環境變數的參數傳進去。那這樣我在 flask-demo 的 container 裡面，要去運行 os.environ.get('REDIS_HOST') 的時候，他的環境變數就會抓到 redis-server 這個 container name，就等於抓到 ip，然後再配上後面的 port 已經寫好 6379，那就可以去 redis-server 裡面取資料了

1.2. Dockerfile

  • Dockerfile 的內容如下：
FROM python:3.9.5-slim

RUN pip install flask redis && \
    mkdir /src && \
    apt-get update

COPY app.py /src/app.py

WORKDIR /src

ENV REDIS_HOST=redis

EXPOSE 5000

CMD python app.py

2. image 準備

  • 從這邊開始，就都是在 AWS EC2 linux server 裡做事了
  • 首先，把 Filezilla 連到 AWS EC2 linux server，並把剛剛的專案資料夾複製過去
  • 再來，從 terminal ssh 到這個遠端 server
ssh -i "./desktop/aws_pair_key/shiny_deploy.cer" ubuntu@ec2-3-15-41-154.us-east-2.compute.amazonaws.com
  • 接著就可以開始做事了

2.1 flask-demo

  • 我們把剛剛寫好的 Dockerfile 做成 image，並取名為 flask-demo 
cd home/ubuntu/flask_redis
docker image build -t flask-demo .

2.2 redis

  • redis 的話，就直接從 dockerhub 拉下來就好
docker image pull redis

3. docker bridge 網路準備

  • 我們建立一個 bridge 網路，名稱叫他 demo-network
docker network create --driver bridge demo-network

4. 建立 container

4.1 建立 redis container

  • 建立一個叫 redis-server 的 container
    • 裡面用的 image 是直接從 dockerhub 拉下來的 redis
    • 網路要用剛剛定好的 demo-network
docker container run -d --name redis-server --network demo-network redis

4.2 建立 flask container

  • 建立一個 flask 的 container
    • 裡面的 image 是剛剛 build 好的 flask-demo
    • 網路要用剛剛定好的 demo-network 
docker container run -d --network demo-network --name flask-demo --env REDIS_HOST=redis-server -p 5000:5000 flask-demo
  • 這邊注意到，有用到 --env REDIS_HOST=redis-server 這個指令，就是把環境變數傳到這個 container 裡面，那現在這個 container 中，就可以調用 REDIS_HOST 這個環境變數
  • 對照到剛剛的 app.py，現在 redis = Redis(host=os.environ.get('REDIS_HOST', '127.0.0.1'), port=6379) 這行指令，就REDIS_HOST 這個環境變數就可以讓他抓到 redis-server 這個 container 的 ip了 (至於後面的 127.0.0.1 不用理他，那個是當你環境變數抓取失敗時，他會 assign 這個 ip 當替代方案）
