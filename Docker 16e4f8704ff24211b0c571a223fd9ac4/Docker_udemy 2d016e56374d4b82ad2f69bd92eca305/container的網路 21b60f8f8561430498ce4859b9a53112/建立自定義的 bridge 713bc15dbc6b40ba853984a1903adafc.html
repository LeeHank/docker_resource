建立自定義的 bridge

# 查詢一下 network 可用的功能
docker network --help # 可以看到有 ls, inspect create, rm, prune, connect, disconnect
# 前面的 ls, inspect 之前學過了，今天要來學 create, cnnect, disconnect


# 新增網路
docker network create -d bridge mybridge # -d 是 driver 的意思, 後面跟 bridge，表示要 bridge網路，最後跟一個你幫這個網路命名的name

# 建立 container 時，指定要連這個網路
docker container run -d --network mybridge --name box3 busybox # --network mybridge 就是設定這個 container 的網路要是 mybridge

# 幫某個 container，開通別的 bridge 網路(注意：一個 container 可以被開通多個網路)
docker network connect docker0 box3 # 用 connect，幫 box3 這個 container，開通 docker0 這個網路
# 幫某個 container 關閉連結
docker network disconnect docker0 box3

# 刪除網路
docker network rm mybridge

  • 但實際上，我們也可以建立自己的 bridge
  • 我們想做出以下這張圖的配置：
  • 從圖中可以看出，我的 linux 主機的 ip 是 192.168.200.10
  • 然後，預設的 bridge 網路，叫 docker0 ，他的 subnet 是 172.17.0.1 (可用 docker network inspect docker0 看到 subnet 的資訊)
  • box1 和 box2 是我之前開過的 container，那時候沒有指定網路，所以預設就都是連到 名稱叫做 docker0 的 bridge 網路。那他們被分配到的 ip 分別是 172.17.0.2 和 172.17.0.3
  • 那今天，我想要自己建立一個新的 bridge 網路，叫做 mybridge ，我可以用以下指令：
    docker network create -d bridge mybridge
  • 裡面的 -d 是 driver 的意思，你也可以寫的完整點，寫成 --driver 。後面就是跟網路 driver 的類型(bridge/host/null)，最後加上這個字定義網路的名稱 (mybridge)
  • 接著，我開一個 container 時，就可以指定他連到 mybridge。用的指令是：
    docker container run -d --network mybridge --name box3 busybox
  • 這邊只有 --network mybridge 是新的指令，表示我要指派這個 container，他的 network，要開通到 mybridge 這個網路。而其他的 -d 是之前教過的 dispatch 的意思，背景執行。 --name 是 container 名稱的意思，我叫他 box3。
  • 那目前為止，這個 container 已經連到 mybridge 這個網路了。我們可以用 docker container inspect box3 來看到他的連線類型是 bridge，然後 ip 也在 mybridge 的 subnet 之中
  • 最後，這邊提醒一下，同一個 container，可以開通多個網路。所以，我如果希望 box3 這個 container 可以和 box1 & box3 通訊，那我就需要將 box3 也加入到 docker0 這個 bridge network 中，這時候我可以用以下指令：
    docker network connect docker0 box3 # 用 connect，幫 box3 這個 container，開通 docker0 這個網路docker network connect docker0 box3 # 用 connect，幫 box3 這個 container，開通 docker0 這個網路
  • 一樣，我們再檢視一下 box3 這個 container 的網路狀況( docker container inspect box3 )，就可以發現在 network 下面，他有連到 docker0 和 mybridge 這兩個網路。
