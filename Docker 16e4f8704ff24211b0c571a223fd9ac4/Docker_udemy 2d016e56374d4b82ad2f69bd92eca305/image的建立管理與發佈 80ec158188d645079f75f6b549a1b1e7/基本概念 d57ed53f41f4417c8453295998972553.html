    • 法二：從我的電腦的 dockerfile 中， build 一個 image 出來
    • 法三：從我的電腦中的某個 local 資料夾， load 一個存好的 backup.tar 文件

1. 從 docker registry 中 pull image 下來

  • 這是最省力的做法
  • docker registry可以分成
    • public: 公開的雲端空間，讓你下載 image，例如 dockerhub
    • internal: 公司內部的雲端空間，通常都是先經過資安程序，把常用的image，先拉到公司自己的雲端空間後，你再去公司的這個雲端空間拉你要的 image。
  • 作法上，其實之前 hahow 那邊的課都學過了，看 這裡 就好

2. 把 image 寫成 .tar 檔後，用 load

  • 這邊的情境是這樣，如果你今天的工作環境是不能上網的，那就無法用 dockerhub 之類的方式來取得 image。
  • 那簡單的想法就是
    • 我先在一台可以上網的電腦，去把我要的 image 給 pull 下來。
    • 然後，我把這個 image 寫成一個檔案。
    • 接著，我用隨身碟之類的方式，把這個檔案，移到那台不能上網的電腦上
    • 最後，我就在這台不能上網的電腦上，用 load 的方式，去使用這個 image

4. 在 container 操作完一切後，再 commit 出 image

  • 這個方法也蠻少用的，但蠻酷的，紀錄一下。
  • 假設我今天開一個 container ，裡面跑的 image，就只是單純的 ubuntu
  • 那我進到這個 container 後，我可以去安裝 python，並新增一個 "hello.py" 的檔案
  • 那如果我希望目前這個環境 (ubuntu + python + 一個 hello.py 檔案)，可以包成 image，除了自己寫一個 dockerfile 外，我還可以用 commit 的方式，把目前的 container 裡面的環境給 commit 出一個 image
  • 作法如下：
    • 先開一個 container ，用 interactive 模式，進去這個 container 裡 (e.g. docker container run -it ubuntu sh )。這邊提醒一下，沒有說一定要在這邊就用 interactive，你也可以先開 container 後，再用 docker exec -it {container_id} 進去。
    • 在 shell 中，安裝 python，新增 hello.py 檔案
    • exit, 退出 interactive 模式
    • 這時，這個 container 的狀態，是停止的，但沒有被刪除，你用 docker container ls -a 可以看到這個 container id。
    • 最後，做 
    docker container commit {container_id} {def_image_name}
# 例如： docker container commit 75e035739753 cktottti/my_python_demo
    • 這樣就完成了。那這樣做的缺點是，你只是把環境寫成 image。沒辦法像 dockerfile 一樣，不只把環境包好，還可以寫接下來要做啥 (e.g. 最後要做 python hello.py )
