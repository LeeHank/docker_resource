2. 什麼是 container

  • container 的全名，其實是 Linux container (LXC)，他是在 linux 系統中誕生的一項技術，為一種內核輕量級的作業系統層虛擬化技術。所以，你可以用 Docker 去執行這項 LXC，也可以用 k8s 去執行這項 LXC，都可以。重點是在LXC。這是 linux 的技術。
  • 那 LXC 主要由 Namespace 和 group 兩大機制來保證實現：
    • Namespace: 命名空間，用於資源的隔離。(就像虛擬機，彼此獨立)
    • Cgroup: 負責資源管理控制作用，比如進程組使用 CPU/Memory 的限制，進程組的優先級控制，進程組的掛起和恢復等。(這就和 VM 不同了，VM是定死 CPU/Memory，所以你開很多VM時，電腦就準備掛了。但用 container 的話，就好像動態幫你開 VM 一樣， CPM/Memory 會幫你自動配置。)
  • container 是一種快速的打包技術，具有：
    • 標準化：2015年，由 google, docker, redhat 等廠商，聯合發起 Open Container Initiative (OCI)，致力於容器技術的標準化。內容包括：
      • 容器運行時的標準：規定了容器操作的規範。例如：如何下載 image, 創建 container, 啟動 container 等。
      • 容器 image 的標準：定義 image 的基本格式。那這樣你用 docker/podman/k8s 建立的 image, 都可以被 container 使用。
    • 輕量級：跟 VM 比，輕量很多
    • 易移植：和標準化有關，因為現在都標準化了，所以他很方便的移到不同的電腦主機上(跨OS, 跨主機)。container 就像海運時用的集裝箱，他很容移植到別的港口。
