Get started

docker-compose # 可以查到後面可以下的指令，e.g. up, run, start, stop, rm,...

# 啟動容器(服務)
docker-compose up
docker-compose up -d # 背景執行

# 查看目前狀況
docker-compose ps # 注意，沒有 ls 這個指令，用 docker-compose ls 是行不通的

# 暫停
docker-compose stop # 停止所有用 docker-compose 指令所創建出來的容器s

# 刪除
docker-compose rm # 刪除用 docker-compose 指令所創建出來的容器s

1. 基本語法結構

version: "3.8"

services: # [必填] 容器s
  servicename: # [必填] 第一個容器的名字，這個名字也是自定義 bridge 網路可 ping 通的 DNS name
    image: # [必填] 要使用的 image 的名字
    command: # [選填] 如果有填，會覆蓋 image 裡寫過的 CMD 指令
    environment: # [選填] 相當於 docker container run 裡的 --env
    volumes: # [選填] 相當於 docker container run 裡的 -v
    networks: # [選填] 相當於 docker container run 裡的 --network
    ports: # [選填] 相當於 docker container run 裡的 -p
  servicename2:

volumes: # 可选，相当于 docker volume create

networks: # 可选，相当于 docker network create
  • 最上面的 version ， 是指 docker-compose 的 version，可以到官網的這裡參考：https://docs.docker.com/compose/compose-file/ ，簡單截圖如下：

2.1 Python flask + Redis 練習

  • 先看一下，之前從頭到尾的作業流程長這樣：
    1. 建立專案資料夾 flask_redis ，並切到這個資料夾下面 ( cd home/ubuntu/flask_redis )
    1. 在這個資料夾中，寫好
      1. app.py : 這是個 flask 的應用
      1. Dockerfile： 用來生成 flask-demo 這個 image 用的
    1. 開始在 terminal 中，依序下達以下指令：
      # 確認正在專案資料夾下
pwd # 可以看到正在 home/ubuntu/flask_redis
ls # 可以看到 app.py Dockerfile

# image 階段 ----
docker image build -t flask-demo .
docker image pull redis

# network 階段 ----
docker network create --driver bridge demo-network

# container 階段 ----
docker container run -d --name redis-server --network demo-network redis
docker container run -d --name flask-demo --network demo-network  --env REDIS_HOST=redis-server -p 5000:5000 flask-demo
  • 那如果寫成 docker-compose，步驟如下：
    1. 建立專案資料夾 docker-compose-demo ，並切到這個資料夾下面 ( cd home/ubuntu/docker-compose-demo )
    1. 在這個資料夾中，寫好
      1. app.py : 這是個 flask 的應用
      1. Dockerfile： 用來生成 flask-demo 這個 image 用的
    1. 在 terminal 中，先把 image 和 network 準備好
      # 確認正在專案資料夾下
pwd # 可以看到正在 home/ubuntu/docker-compose-demo
ls # 可以看到 app.py Dockerfile

# image 階段 ----
docker image build -t flask-demo .
docker image pull redis

# network 階段 ----
docker network create --driver bridge demo-network
    1. 在專案資料夾中，新增一個 docker-compose.yml 文件內容如下： (補充：yaml檔的檔名一定要叫 docker-compose.yml ，到時候你下 docker-compose xxx 的指令時，他才抓得到。概念上，就像 Dockerfile也是只能叫這個名字。)
      version: "3.8"

  flask-demo: # 第一個 container 的 container name

    networks: # 使用的網路name
      - demo-network
    ports: # port mapping
      - 5000:5000

  redis-server:
    image: redis:latest
    networks:
	    - demo-network

networks:
  demo-network:
