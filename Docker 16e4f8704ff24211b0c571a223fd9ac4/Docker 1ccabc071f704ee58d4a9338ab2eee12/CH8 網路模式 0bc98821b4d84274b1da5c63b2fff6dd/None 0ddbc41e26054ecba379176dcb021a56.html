None

  • 今天要介紹 docker network裡的 none 模式

docker network ls # 列出目前所有的
docker network inspect none # 看這個名字的network的內容
docker exec -it {container_id} /bin/sh # 進到正在執行的container裡面，用互動模式，的shell
ip addr ls # 列出目前的 ip

  • 列一下目前所有的 docker network: docker network ls ，得到以下結果：
  • 看一下 DRIVER那欄，有bridge, host, null三種模式。而null就是我們這邊說的none模式，也就是沒有網路功能的模式。
  • 實驗開始，我們現在先抓一個image下來做demo: docker pull alpine
  • 我們現在開一個container，讓他去 run 剛剛的 alpine 這個 image。語法如下：
    docker run -d --network none --name none-mode alpine tail -f /dev/null
docker container ls # 看一下有在跑的 container 有哪些
    • -d 讓他背景執行
    • --network none 今天的重點。--network 表示我要下 network的指令，後面接 NAME (上圖用 docker network ls所看到的NAME)
    • --name none-mode 幫這個 container取個名字而已，不是必要的。但有寫名字的話，之後可以用名字來call他，不用去複製這個container id，是也會比較方便。
    • apline 就我要run的Image(輕量版linux)，沒什麼。
    • tail -f /dev/null 讓這個linux去跑的一個指令，讓他一直有事做，不要斷掉用的而已。
  • 接下來，我們用 docker network inspect none ，來檢查現在這個 none 的 network 的內容
    • 可以發現，這個network裡面列了一個container，名稱就是剛剛我開的container的ID，表示我把這個container，assign 到這個 none network裡面了。
  • 最後，我們進到這個container裡面，看看他能不能連上網路(果然不行)：

  • 額外補充，今天介紹的 none 模式，也是開container時的預設模式。也就是說，我如果開container時，用的 code 如下：
docker run -d alpine tail -f /dev/null
  • 從上面的code，可以看出我沒有指定網路模式。那這時的網路模式就是none
  • 所以這個container，現在就是無法連網路的(內外網都不行)。
