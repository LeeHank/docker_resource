  • 我們這邊來設想一個用docker的情境： 我的電腦不是linux，而我想使用linux的一些功能，但我又不想開VM來灌linux，那...我就把linux的image抓下來，然後開一個docker的container，去run這個linux就好。
  • 這個例子當然很詭異，畢竟你要run docker的話，勢必要先用VM開linux才能用。但現在先別糾結在這，這就是一個簡單的範例而已。
  • 我們首先切到這章的目錄，並新增一個Dockerfile:
  • 用VSCode打開這個Dockerfile，然後進行以下編輯：
  • 可以看到，Dockerfile中只有一行指令： FROM alpine:latest(特別注意，中間不要有空格，不要打成alpine: latest)
  • alpine就是個輕量化的linux
  • 有了這個Dockerfile後，我們就可以來build image了：docker build -t cktottti/001 .。這邊的t是加上tag的意思，也就是幫我們的image取名字。這邊取名為cktottti/001，前面的cktottti/是docker hub的帳號名稱。最後的.，是指要build image時，去哪個資料夾找Dockerfile。而因為Dockerfile就在我們目前的目錄中，所以打.即可。
  • 可以看到，現在已經多了cktottti/001這個image，而這個image其實就是alpine。
  • 有了image後，我們就可以來使用看看了。也就是開個container來跑他：docker run cktottti/001 cat /etc/os-release
  • 這行指令的docker run，其實是docker container run的簡寫，他就是要docker開一個container，來run的意思。那run什麼呢？就是run後面的image: cktottti/001。那目前這個image，其實就是一個alpine，就是一個輕量的linux os，所以我run這個image的意思，就好像我進入到一個linux server。那進入後要做啥呢？我要做cat /etc/os-release這件事，也就是我想看我的作業系統版本：
  • 這邊提醒一下，我們run一個image時，他會開啟一個container來run他，但一run完，他就把container清空了。所以剛剛我們做的cat /etc/os-release，其實只是去看看版本而已，所以一執行完他就清掉這個container，
  • 我們用docker container ls去看現在有哪些container在運行，
  • 可以發現，現在沒有container在運行
  • 那我們現在試試讓container一直有事情做，讓他停不下來： docker run -d cktottti/001 tail -f /dev/null，其中，d是daemon的縮寫，意思是要在背景執行。tail -f的意思是，要去追蹤(-f是指follow的意思)某個檔案，並把這個檔案的內容print在console。哪個檔案呢？/dev/null這個檔案。
