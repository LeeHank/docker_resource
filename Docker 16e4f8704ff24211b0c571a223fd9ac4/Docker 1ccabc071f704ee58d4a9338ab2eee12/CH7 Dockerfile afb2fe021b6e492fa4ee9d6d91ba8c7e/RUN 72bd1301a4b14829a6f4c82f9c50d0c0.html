RUN

  • RUN和ENTRYPOINT的最大不同，就是他是在build image時，就要執行的指令。
  • RUN這個指令，通常都接在FROM的下面，他的功能，是去使用任何base image提供的指令。所以以我們的R為例，一開始的FROM當然就是去灌作業系統，例如centOS，那接下來的RUN就是要去安裝我們的R和RStudio套件，所以會寫成RUN yum install blablabla...
  • 那我們現在先延續剛剛最簡單的例子，因為現在的base image是alpine(i.e. linux)，那我試試看在Dockerfile中加上RUN echo "I'm in the image building process..."(在console上print出"I'm in the image building process...")和RUN ls -l /(列出根目錄中的檔案)，是不是就能在建image時，執行這兩個指令：
  • 可以看到，在建立image時，藍色字的倒數第七行開始，就看到CACHED [1/3] FROM ...在安裝base image, [2/3] RUN ...在做echo， [3/3] RUN ... 在做ls。所以可以看到RUN和ENTRYPOINT的差異：RUN是建立image時，要執行的指令(此指令是使用base image提供的指令)。ENTRYPOINT是啟動container後，要執行的指令。
  • 那接下來，我們就來做個實例，我們的目標是：
    • 在alpine linux中，下載apache web server
    • 在container中，開啟appache服務，做好port mapping後，於local主機輸入local port，就連到container的port，進而成功開啟網頁
  • 我們先示範笨拙版(使用cktottti/002這個只灌好linux的image，然後接著自己在terminal中做完上面的目標)，然後再示範Dockerfile版(把上面所有事都包成image，那我只要一run這個image，就完成所有事了)。
  • 首先是笨拙版：
  • 可以看到，我先run這個image，但加入一些之前沒看過的指令：
    • d：讓他在背景執行，不要停下來。d就是daemon，把它想成docker daemon，就是在背景去run的意思。
    • p 8080:80: p是port mapping的意思。然後docker有三個空間，第一個是實體機mac，第二個mac上裝的虛擬機linux ，第三個是虛擬機上跑起來的container。那這邊的8080，是指虛擬機linux的port，這個可以自己亂設，你要叫5566也可以。80是這個container上的port(i.e. alpine上的port)，這就不要亂設，apache的預設就是80，就像RStuio server的預設是8787，這都不要改比較好。
    • 有了這個對應後，我去虛擬機上的8080，就會連到container上的80。等等就會有範例。
  • 接下來，我進到container中，去安裝apache web server，並開啟他：
  • 開啟apache服務後，我們到我們的瀏覽器，key入： localhost:8080，他就會從8080 port，幫我們mapping到container的80 port，也就是apache提供服務的地方，就能順利開啟這個網頁：
  • 總結一下，我們剛剛的笨拙版本，共用了以下的語法：
docker run -d -p 8080:90 cktottti/002
docker exec -it {container id} /bin/sh
apk --update add apache2
rm -rf /var/cache/apk/*
httpd -d FOREGROUND
  • 那我現在想把這些行為，都寫到Dockerfile裡面，就可以寫成這樣：
  • 從Dockerfile中可以看到
    • 我們先下載alpine linux
    • 然後用alpine linux中的指令：apk ...去安裝apache2
    • 用apline linux中的指令rm -rf ...清掉暫存檔
    • 到此為止，包好一個image了。所以這個image是有linux，且linux上灌好了apache這個套件。
    • 接著，在container起起來這個image後，就要執行ENTRYPOINT的指令： httpd ....，也就是在container中，開啟apache服務。
  • 再來，我們就去建立這個image: docker build -t cktottti/004 .
  • 最後，我只要在terminal中，key入 docker run -d -p 8081:80 cktottti/004，他就會依序做完：
    • 開啟一個已經灌有apache的alpine linux
    • 打開他的服務(ENTRYPOINT ["httpd", "-d", "FOREGROUND"])
    • 所以，我們在本地的瀏覽器，key 入 localhost:8081就會看到結果
