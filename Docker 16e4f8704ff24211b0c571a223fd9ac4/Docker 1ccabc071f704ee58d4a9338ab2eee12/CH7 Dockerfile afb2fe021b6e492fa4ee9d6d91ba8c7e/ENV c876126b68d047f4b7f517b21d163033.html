ENV

  • ENV是環境變數的縮寫，他在Docerfile中的角色，就是幫你命名成一個變數。
  • 舉例來說，如果我在Dockerfile中，要頻繁地使用到以下的路徑名稱： "/var/www/localhost/htdocs"，那我就可以在Dockerfile中，寫成：ENV myworkdir /var/www/localhost/htdocs，其實意思就像是R/Python在命名變數的： myworkdir = "/var/www/localhost/htdocs"。那我後續要用這個變數時，我得這樣用${myworkdir}，而不是單純的寫myworkdir而已。
  • 我們以下舉一個例子來說明
  • 首先要釐清一下觀念，當我們用Dockerfile在build image時，他是每執行一步，就包成一個暫時的image，所以行與行之間是獨立的
  • 例如，Dockerfile長成以下這樣：
FROM alpine:latest
RUN cd /var/www/localhost/htdocs
RUN pwd
  • 那，他在build image時，是執行RUN cd ...，就包成一個image，所以，當你再執行下一行RUN pwd時，他根本不記得上一行的指令了，所以不會print出 "/var/www/localhost/htdocs"
  • 那，簡單的寫法，就是把RUN都擠在同一行，然後用&&來連接：
FROM alpine:latest
RUN cd /var/www/localhost/htdocs && pwd
  • 如果要版面好看一點，我們可以用反斜線\\來換行，就寫成：
FROM alpine:latest
RUN cd /var/www/localhost/htdocs \
    && pwd
