2. pull image方式

  • 要記得在dockerhhub的image前面，加上harbor.wnc.com.tw/dockerhub，例如

docker pull harbor.wnc.com.tw/dockerhub/asachet/rocker-tidymodels

3. 重寫 Dockerfile

  • 在以下路徑有範本: /home/hadoop/EDSA/dockerR_Hank
  • Dockerfile的內容如下：
FROM harbor.wnc.com.tw/dockerhub/rocker/rstudio:4.0.4
# FROM harbor.wnc.com.tw/dockerhub/asachet/rocker-tidymodels

#設定 proxy
ENV http_proxy='http://172.16.100.51:3128'
ENV https_proxy='http://172.16.100.51:3128' 

#安裝 renv，指定 cloud.r
RUN R -e 'install.packages("renv",repos="https://cloud.r-project.org")'

#設定 Java參數
ENV JAVA_HOME=/usr/java/jdk1.8.0_152
ENV JAVA_JRE="/usr/java/jdk1.8.0_152/jre/bin"
ENV PATH="/usr/java/jdk1.8.0_152/bin:$PATH"
ENV PATH="/usr/java/jdk1.8.0_152/jre/bin:$PATH"
ENV JAVA_CPPFLAGS="-I/usr/java/jdk1.8.0_152/include -I/usr/java/jdk1.8.0_152/include/linux"
ENV JAVA="$JAVA_HOME/bin/java"
ENV JAVA_LIBS="-L/usr/java/jdk1.8.0_152/jre/lib/amd64/server -ljvm"
ENV LD_LIBRARY_PATH=$JAVA_HOME/jre/lib/amd64:$JAVA_HOME/jre/lib/amd64/server:/usr/local/lib


#設定 renv global cache
RUN   echo "RENV_PATHS_CACHE=/shared_package" >> /usr/local/lib/R/etc/Renviron

#設定使用者
RUN useradd -u 1004 -ms /bin/bash tom-hanks -p 19002329
RUN echo 'tom-hanks:19002329' | chpasswd

#設定 proxy 到 contain 的 rstudio
RUN   echo "http_proxy='http://172.16.100.51:3128'" >> /usr/local/lib/R/etc/Renviron
RUN   echo "https_proxy='http://172.16.100.51:3128'" >> /usr/local/lib/R/etc/Renviron

#修改 repos 預設
COPY Rprofile.site /usr/local/lib/R/etc/Rprofile.site

#不能安裝 java，所以直接搬本機的過去
COPY java /usr/java

CMD ["/init"]

  • 這邊特別要題的是 設定使用者
    • 我要先去putty，查一下我在hadoop測試區裡面，tom-hanks的uid是多少。call cat /etc/passwd ，就會看到 tom-hanks 的 uid 是 1004
    • 所以，我在dockefile裡面，先去新增container裡面的user是1004，帳號密碼也設好，如以下： RUN useradd -u 1004 -ms /bin/bash tom-hanks -p 19002329
    • 然後加上網路上查的這組帳密寫法(原理我還不清楚) RUN echo 'tom-hanks:19002329' | chpasswd
    • 那這樣，我進到 container 裡面後，這個linux裡面就已經存在 tom-hanks 這個帳號了。
    • 至於為啥要設定這個。因為之後都會把container裡面的資料夾 mount 到測試區server中，這時候我在container裡寫的東西，他的預設權限都是 container 裡的 userid (uid)
    • 所以，我如果沒有去設定剛剛1004的這些東西，那預設的使用者是rstudio，他的uid是1000，那你就會在mount出來的hadoop測試區資料夾下，看到寫出來的東西的權限是1000，你完全無法從外部做更動了
    • 相反的，因為我現在有在container裡面設好uid了，寫出來的時候，就會是和本機一樣的 tom-hanks權限，這樣我就很好處理了
  • 也要提一下 修改 repos 預設
    • repos 是告訴R要下載package時，要去哪下載。例如我們在 R console裡面打 getOption("repos")，就會show出預設的下載路徑：。但這個路徑會擋，所以我們想把這個路徑換到 https://cloud.r-project.org。
    • 那作法就是，打開hadoop測試區裡的 /usr/local/lib/R/etc/Rprofile.site ，裡面長下圖這樣，我去改第一行就好
    options(repos = c(CRAN = 'https://cloud.r-project.org'), download.file.method = 'libcurl')
options(HTTPUserAgent = sprintf("R/%s R (%s)", getRversion(),
                 paste(getRversion(), R.version$platform,
                       R.version$arch, R.version$os)))
    • 然後我把這個改好的檔案，和dockerfile放在同一個資料夾。等等建image時，就把這個檔案，copy到container裡面的相同位置，取代掉原本的Rprofile.site，這樣就搞定了

  • 然後登進去RStudio的console裡面，call `renv:::

5. 跑 image

docker run --privileged -ti --user root \
-v /shared_package/v5/R-4.1/x86_64-pc-linux-gnu:/shared_package/v5/R-4.1/x86_64-pc-linux-gnu \
-v /home/hadoop/docker_r_projects:/home/tom-hanks/docker_r_projects \
-v /home/dmis-aaron:/home/dmis-aaron \
-v /home/hadoop/EDSA/Def:/Def \
-d -p 8877:8787 \
tomhanks/tidymodels
  • 這邊要注意幾點
  • 第一個 -v ，是去對應hadoop測試區的shared_package(因為image是灌R-4.0，所以要先去這個位置新增R-4.0的資料夾)，和container裡面的shared_package
  • 第二個 -v ，是我會把r project都放到docker_r_projects這個位子下，那在container裡面新增資料夾(e.g. 這邊的revenue)，就可以對回hadoop的資料夾，在container裡面做的事情，就都匯存出來了
  • -e HTTP_PROXY 這個應該不用寫了，因為寫在dockerfile裡面了

Optional: 進到container裡面

  • 先用 docker container ls ，去找到正在 run 的 container id，e.g. d8e79fe942b0
  • call docker exec -i -t d8e79fe942b0 /bin/bash 就近到這個container的bash裡面了
  • 此時的身分是 root ，所以要新增或刪除檔案都是無敵星星。
