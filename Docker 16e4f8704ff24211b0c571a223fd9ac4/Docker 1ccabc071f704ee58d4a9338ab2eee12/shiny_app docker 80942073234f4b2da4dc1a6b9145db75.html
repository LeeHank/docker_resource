shiny_app docker

  • 這邊記錄一下做法

1. 照往例寫 app

  • 先照往例，寫好一支 app (renv管控)，例如這個AP /srv/shiny-server/sleep_app/
  • 然後，先把這支 app 的 renv library，從 link到global cache的方式，轉換成全都塞實體package檔案： renv::isolate()

2. 將 app 包成 docker image

  • 開個資料夾(e.g. /home/hadoop/EDSA/dockerR)，裡面放 dockerfile, java資料夾
  • dockerfile 主要是安裝 linux + R，並把 java 給 copy 進去。只是做好這個環境而已 (至於第一點的 app，之後是用 mount 的方式掛上去)
  • 看一下這個 dockerfile

  • 然後，就把這個 image給 build起來：
cd /home/hadoop/EDSA/dockerR
docker build -t sleep_test .
  • 那現在，就有個 sleep_test 這個 image了

3. 去 shinyproxy 的 config 檔做設定

  • 那之前包好 image 後，就會開個 container 去跑他。那因為現在是 shiny 的應用，所以只要去 shinyproxy 的 config 檔做設定，然後重啟 shinyproxy，他就會自動幫你開 container 去跑裡面定義好的 images
  • 在 terminal 輸入以下訊息，可以打開 shinyproxy 的 config 檔：
sudo vim /etc/shinyproxy/application.yml
  • 那內容會長這樣：

  • 可以看到，要改的只有這裡：
  - id: SleepTest
    container-cmd: ["R", "-e", ".libPaths('/root/sleep_app/renv/library/R-3.4/x86_64-redhat-linux-gnu');shiny::runApp('/root/sample_size/',port=3838,host='0.0.0.0')"]
    container-image: sleep_test
    container-volumes: [ "/srv/shiny-server/sleep_app:/root/sleep_app/" ]
  • 第一行的 id，就是你的 shiny app 要叫的名字，我取 SleepTest，那之後就是用 http://172.16.99.233:8082/app/SleepTest 來開啟連結 (ip是測試區ip，因為 shinyproxy 是架在測試區。port號 8082 是在 config 檔最上面設定的)
  • 第三行的 container-image 就是放剛剛包好的 image
  • 第四行的 container-volumes 就是在 mount資料。我把 linux 測試區上的 /srv/shiny-server/sleep_app 這個資料夾，和 sleep_test 這個 image 裡的 linux 的 /root/sample_size/ 這個資料夾連動
  • 第二行，就是我在這個 container 裡面想下 command，我想指定我的 R 的 library 要去哪個路徑拿資料：我要去 /root/sleep_app/renv/library/R-3.4/x86_64-redhat-linux-gnu 這個路徑下取 package。然後，我要啟動我的這支 app，所以我下 `shiny::runApp('/root/sample_size/',port=3838,host='0.0.0.0')` 這個指令。

4. 重啟 shinyproxy

sudo systemctl restart shinyproxy

5. 開啟服務 & 取登錄者資訊

  • 在瀏覽器上輸入 http://172.16.99.233:3838/sleep_app/ 即開啟服務
  • 在 R 裡面用 Sys.getenv('SHINYPROXY_USERNAME') ，即可取得登入者資訊。
