CH2: 在不同OS上使用Docker

2.0 本章節所有程式碼匯總

# 安裝docker  
cat /etc/os-release # 查看/確認目前的os版本資訊
sudo yum install docker -y # yum是centos的裝法 -y指同意所有default設定

# 開啟服務
sudo service docker start

# 示範使用docker image
sudo docker images # 看local端有哪些docker images，發現沒有
sudo docker pull hello-world # 去docker hub抓hello-world這個image  
sudo docker images # 確認local端已有hello-world這個image
sudo docker run hello-world # 會去執行本地的hello-world這個image  

# 關閉服務
sudo service docker stop

2.1 Intro

  • 這章會簡單示範一次，分別在Linux, Mac, Windows上，如何使用docker

2.2 觀念講解

  • 上一章有講過，docker的意思，就是在現有的OS上，架一個docker engine，有了這個docker engine後，就可以開container，然後把docker hub上面的image下載下來，丟到container中去執行
  • 所以，這一章說的：如何使用docker，其實就在教如何架一個docker engine
  • 先講結論：docker engine只能架在linux上，所以如果是mac/windows，就要先開個虛擬機(VM)，在這個虛擬主機上安裝linux，然後進入這個linux後，才開始架docker engine
  • 講師的圖解概念很清楚，請看下圖：
  • 可以注意到，圖形的下半部，mac/windos都要透過virtual Box來建立linux後，才執行docker
  • 那…我原本以為，左邊那條路(Virtual Box)，就是我之前在上linux的課時，直接用Oracle VM VirtualBox這個軟體，在Mac上安裝一個centOS7的虛擬機，然後我們再用這個虛擬機來執行docker。那當然這樣也沒問題，就像真的在linux上做事，而如果你要使用檔案的話，也只能使用linux主機上的檔案(換句話說，如果你想執行從別人那邊下載過來的Dockerfile，那得先把這個檔案，從自己的mac/windows，傳到linux主機上的資料夾，才可以開始run)
  • 但其實這張圖的左路的Virtual Box，是指docker幫你做好的虛擬機(可去Docker網站上找Docker Toolbox來下載)，安裝完後，會有他專屬的terminal，然後一打開terminal，就可以執行docker的指令了。所以可以想成這個toolbox提供給你的terminal，就是一路幫你串到linux後所打開的shell，所以你可以繼續用自己的mac/windows的檔案系統做事，這就比我原先設想的方便多了
  • 那對Mac用戶來說，右邊那條路就更友善了(Docker網站上有提供Docker Desktop可以下載)。只要安裝完Docker Desktop，那打開此應用程式，他就會有dashboard讓你看你現在有哪些container, image等訊息，而且，自動幫你連線好，此時，只要打開原本的mac shell，就可以直接執行docker指令，不需要再去用剛剛左路的那種特製terminal了
  • 所以，以下的節次，會先示範linux的做法(我就在mac上的centOS7虛擬機上操作，來模擬一般在公司時直接操作linux的情境)，然後示範mac上用Docker desktop的做法。至於windows，我就先懶惰不整理了。
  • 之後的章節，則一併用mac docker desktop的模式做示範。

2.3.1 Linux示範

  • 以下的操作，是我進入我的mac上的虛擬機：centOS(帳號：tom-hanks; 密碼：19002329，有灌UI的版本)，所做的操作。可以直接類比成在linux主機上的操作

2.3.1.1 安裝docker

  • 首先，使用cat /etc/os-release查看/確認一下目前的os版本
  • 接著，使用sudo yum install docker -y，安裝docker。注意，y是指同意所有預設安裝步驟，每個步驟就別再問y/n了
  • 因為執行後的畫面頗多，我就只擷取最前面的結果，反正最後就是complete!

2.3.1.2 啟動docker

  • 使用 sudo service docker start 來開啟docker服務。這邊可以學一下linux的寫法，service在linux中是指要啟動某個特定服務，‘這邊的服務就是後面寫的docker

2.3.1.3 跑一個image看看

  • 先看看現在local端有什麼docker images: sudo docker images，發現空空如也
  • 所以，去docker hub下載一個image來玩玩(下載hello-world吧)： sudo docker pull hello-world
  • 下載好後，再確認一次，local端已經有hello-world的docker image了： sudo docker images
  • run一下這個image，看會怎麼樣： sudo docker run hello-world
  • 最後，關閉這個服務： sudo service docker stop
